# storage

Эта папка не хранится в Git целиком. Внутри нее живут runtime-данные:

- `site-content.json` - контент CMS
- `media/` - загруженные изображения
- `contact-submissions.jsonl` - входящие заявки
- `reviews.jsonl` - отзывы и статусы модерации
- `cms-sessions.json` - активные серверные сессии админки
- `backups/` - ручные и pre-restore backup-архивы CMS

Приложение умеет создать эту структуру само при первом запуске, если папка пуста.

Для первого деплоя есть два сценария:

1. Нужен чистый запуск с дефолтным контентом:
   ничего переносить не нужно, приложение само создаст `storage/`.

2. Нужен текущий локальный контент и медиа:
   перенесите содержимое `storage/` на сервер отдельно от Git, например через `rsync`.

Пример:

```bash
rsync -av storage/ user@server:/var/www/perfectweb/storage/
```

На VPS папка `storage/` должна:

- существовать в рабочей директории проекта
- быть доступна на запись пользователю, под которым запущен Node/PM2/systemd
- сохраняться между деплоями
